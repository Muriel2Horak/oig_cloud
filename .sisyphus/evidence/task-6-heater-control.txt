## Control Logic

The heater switch is controlled based on:

1. **Turn ON when:**
   - There is an active slot in the plan (current_slot is not None)
   - Current temperature < target temperature - hysteresis (2.0°C)

2. **Turn OFF when:**
   - No active slot in plan
   - Current temperature >= target temperature

3. **Hysteresis handling:**
   - When temperature is between (target - hysteresis) and target, maintain current state
   - This prevents rapid on/off cycling

4. **State tracking:**
   - Previous state is tracked in self._heater_last_state
   - Service calls are only made when state actually changes
   - All state changes are logged
