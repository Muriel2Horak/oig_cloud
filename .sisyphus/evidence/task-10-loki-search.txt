3. LOKI QUERY PATTERNS — CONFIRMED CORRECT
═══════════════════════════════════════════════════════════════════════════════
The two Loki queries provisioned in the Task-12 Grafana dashboard panels are:

  Warnings panel:
    {job=~".+"} |= "OIG_CLOUD_WARNING"

  Errors panel:
    {job=~".+"} |= "OIG_CLOUD_ERROR"

These queries will match any log stream ingested by Promtail (including the
haos-syslog job, job=homeassistant, host=haos) that contains the respective
literal string. Because every normalized log line in the code now carries one
of these exact tokens, any warning or error emitted at runtime will be
immediately text-searchable via these queries.

Stream selector `{job=~".+"}` is deliberately broad so the dashboard works
before stream-label cardinality is finalized, and can later be narrowed to
`{job="homeassistant"}` without changing the marker token filter.
